# Codex 账号归因与主窗口设计

## 目标

把 Codex 账号管家从“状态栏弹层工具”升级为两层体验：

- 状态栏弹层：快速看当前账号、额度、运行状态、今日 token 估算和切换入口。
- 主窗口 Dashboard：横向透明玻璃界面，集中展示额度、用量趋势、项目排行、工具与 Skill 使用。

## 核心口径

Codex 本地目录共享后，`~/.codex` 的 SQLite 和 rollout 日志不自带稳定账号 ID。因此当天实时 token 不能伪装成官方账号实时值。第一版采用“账号管家归因估算”：

1. 每次账号切换时记录当前 profile、切换时间和本地 token 快照。
2. 刷新时用当前本地快照减去切换时基线，把差值归因到当前 profile。
3. 如果 Codex 不是由账号管家托管，或 active profile 不可信，则归入“未归因”。
4. 第二天官方 `account/usage/read` 返回前一天日桶后，用官方值回填前一天准确数据，并记录估算值和官方值的误差。

这个设计满足当天“看大概、看趋势、看是否异常”的需求；历史日级数据以官方接口为准。

## 信息模型

必须显示：

- 当前账号与托管状态。
- 5 小时额度、7 天额度和重置时间。
- 当前账号可用重置卡数量与最近到期时间。
- 今日 token：优先显示官方当天值；官方当天缺失时显示管家归因估算。
- 近 7 日账号 token：官方账号日桶。
- 本地共享 token 拆分：输入、缓存、输出、推理。
- 项目排行：项目路径、线程数、token、最近活跃。
- 工具与 Skill：使用次数、最近一次使用时间、排序。

不显示：

- access token、refresh token、id token、cookie。
- 未脱敏的唯一 ID。
- 不可验证的“官方实时账号 token”。

## 布局模型

状态栏弹层：

```text
[当前账号 + 状态灯 + 更新时间]
[5h/7d 简表 + 今日归因 token + 重置卡]
[账号切换列表]
[打开主界面 / 刷新 / 重启 Codex / 退出]
```

主窗口：

```text
[标题栏：Logo、当前账号、状态、刷新时间、窗口操作]
[顶部总览：额度圆环 | 今日/近7日/重置卡指标卡 | 托管状态]
[Tab：今日活动 | 用量趋势 | 项目排行 | 工具&Skill]
[当前 Tab 内容区]
```

主窗口采用横向布局，优先避免当前竖版弹层过长、内容拥挤、重置卡明细截断的问题。

## 风格模型

- 风格：macOS glass dashboard，参考 codexU 的透明面板、浅色卡片、清晰数据层级。
- 颜色：低饱和玻璃底，额度绿色/紫色，告警黄/红只用于状态。
- 字体：系统字体，数字用 monospaced digit。
- 约束：装饰层不能遮挡文本；所有长文本必须截断或换行；按钮和图表不挤压。

## 实现边界

v0.8.0 做：

- 切换账本和当天 token 归因估算。
- 官方前一天数据与估算误差对比字段。
- 项目排行、工具排行的本地数据接口。
- 状态栏弹层精简。
- 新主窗口 Dashboard 骨架和四个页面入口。

后续版本再做：

- rollout 事件级归因。
- Skill 精确 token 估算。
- 羊毛进度/成本估算。
