# Codex 工作台账号整合执行契约

## Intent Lock

- 本次只解决：把 Profile Switcher 的成熟能力完整迁入 Codex 工作台，使工作台成为唯一日常 App，同时保留旧 App 冷备。

## Scope Fence

### 范围内

- 最新账号后端与完整 payload。
- 工作台账号协调、运行状态、通知和自动重置。
- 动态菜单栏、账号管理、项目分析、工具与 Skill。
- 登录启动辅助入口。
- 安装、真实切换和视觉验收。

### 范围外

- 删除旧 App、CLI 或账号源码。
- 改变账号认证文件格式。
- 新增外部网络服务或遥测。
- 扩展其他未来工作台模块。

## Approved Behavior

### 必须满足

- 当前登录账号是菜单栏和账号页的唯一主账号语义。
- Python 后端继续是认证、切换和账号状态的唯一权威来源。
- 旧 App 只作冷备，不与工作台并行运行。
- 自动重置保持既有安全条件、幂等和通知能力。
- 切换后必须验证真实登录账号。
- 用户查看并确认迁移成果前不得退役旧 App。

### 明确不改变

- `~/.codex` 与 profile 的账号桥接方式。
- profile 名称和本机登录状态。
- 操作日志 V1.4 的解释和隐私边界。
- Codex 退出 / 重启由既有后端执行的顺序。

## Design Constraints

### 架构约束

- 一个工作台进程拥有唯一刷新协调器。
- 主窗口与菜单栏消费同一快照。
- 业务模型与纯状态机进入 `CodexWorkbenchCore`；AppKit / SwiftUI / UserNotifications 留在 App target。

### 接口约束

- 后端命令限定为现有安全 CLI：`status --json`、`app <profile>`、`consume-reset-credit` 等。
- 新字段全部可选解码；未知字段不得使整个 payload 失败。
- profile 名称继续经过安全正则验证。

### 数据约束

- 最近成功快照只包含脱敏展示字段。
- 旧自动重置偏好域只读写 `automatic-reset.*` 键。
- 安装指纹必须包含实际复制进 App 的两个 Python 文件。

### 依赖约束

- 不新增第三方 Swift package。
- 不直接调用非本机的 ChatGPT 后端接口。

## Task Batches

- Batch 1：冷备基线、后端同步、完整数据契约和安装指纹。
- Batch 2：账号协调、切换验证、运行状态和冷备冲突。
- Batch 3：通知、自动重置与操作日志。
- Batch 4：菜单栏、账号管理、项目和工具页面。
- Batch 5：登录启动、真实安装、双向切换和视觉验收。

## Test Obligations

### 必须验证

- Python 全量测试、Swift Core 全量测试、构建和 verifier。
- 每条 AC 至少映射一项自动化或真实证据。
- 账号切换必须真实执行两个方向并恢复初始账号。
- 菜单栏和三个新增 / 重做页面必须来自新安装产物截图。

### 边界情况

- 缺少额度窗口、未知账号、陈旧缓存、后端错误。
- 最长 profile 名、最大百分比和重置卡列表。
- 旧 App 正在运行。
- 切换命令成功但状态验证不匹配。
- 到期但未耗尽、耗尽但无卡、重复通知与重复幂等键。

### 禁止验证方式

- 不允许为了证明自动消费路径而消耗真实重置卡。
- 不允许用源码断言代替菜单栏和窗口截图。
- 不允许用旧安装包或旧进程截图。

## Review Gates

### 实现前

- `spec → tasks`、`plan → tasks`、风险 → 测试 / 回滚一致。
- 工作树无意外改动，旧冷备产物身份已记录。

### 实现中

- 每批次先失败测试，再最小实现，再相关测试和 focused diff。
- 每个独立回滚单元单独提交。
- 若账号接口、认证、自动重置条件或启动方式发生实质变化，先更新 spec 和契约。

### 实现后

- 全量自动化、安装指纹、进程身份和真实 payload 通过。
- 双向切换恢复初始账号。
- Visual Verdict 和无障碍门禁通过。
- 用户看到迁移成果；旧 App 仍存在且未运行。

## Rewind Triggers

### 回到 spec

- 需要修改认证存储格式、增加网络服务或删除旧 App。
- 无法使用后端权威字段确认当前登录账号。

### 回到 plan / contract

- SwiftUI `MenuBarExtra` 无法可靠呈现动态额度和状态，需要改为 AppKit 状态项。
- macOS 13 的登录启动无法通过当前辅助入口可靠区分手动启动。
- 自动重置共享偏好域与旧 App 不兼容。

### 暂停并询问用户

- 真实切换发现账号冲突或需要重新登录。
- 验收无法恢复开始时账号。
- 必须消费真实重置卡才能继续验证。
- 冷备 App 或凭据文件出现损坏迹象。
