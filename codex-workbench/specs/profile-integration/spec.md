# Codex 工作台账号整合 Spec

## 背景和目标

`Codex 观测站.app` 与 `Codex Profile Switcher.app` 当前同时提供菜单栏入口。观测站只复用了账号后端的一小部分能力，独立账号工具仍承载额度状态、Codex 工作状态、重置卡详情与提醒、自动重置、用量趋势和排行。

本次把这些成熟能力完整迁入工作台，使工作台成为唯一日常 App。现有 Python 账号引擎继续作为唯一账号状态源；旧 Profile Switcher App 在迁移期保留为冷备，验收通过前不得删除。

## 用户场景

### 场景 1：随时查看当前账号

用户只看菜单栏即可知道 Codex 当前实际登录的是哪个账号、首要额度剩余多少，以及 Codex 是运行中、待接手还是空闲。

### 场景 2：快速切换账号

用户从菜单栏或账号管理页选择另一个账号。工作台复用现有安全切换路径退出 Codex、保存刷新后的凭据、接管目标账号、重启并验证真实登录账号。只有验证成功后才提示完成。

### 场景 3：管理额度与重置卡

用户在账号管理页查看两个账号的两档额度、重置时间、逐张重置卡到期时间、提醒和自动重置状态。官方明确额度耗尽时，工作台沿用既有幂等规则自动消费最早到期卡。

### 场景 4：查看工作台分析

用户在独立模块查看账号用量、项目排行、工具调用和 Skill 使用情况，而不是继续打开旧账号面板。

### 场景 5：安全回退

迁移期间如果工作台账号链路失败，用户退出工作台后可以手动打开旧 Profile Switcher 冷备，不需要重新登录账号。

## 范围

- 工作台动态菜单栏状态项和展开面板。
- 完整账号 payload 解码与单一刷新协调。
- 当前登录账号、额度、重置卡、用量、排行和运行状态。
- 安全账号切换与切换后真实验证。
- 重置卡到期通知、自动消费状态机和旧偏好域兼容。
- 账号管理、项目分析、工具与 Skill 页面。
- 手动启动显示窗口、登录启动仅驻留菜单栏。
- Python 后端版本同步、打包和安装指纹。
- 操作日志账号事件与 V1.4 回归。
- 自动化、真实安装、双向切换和视觉验收。

## 非目标

- 不删除或卸载旧 Profile Switcher App。
- 不删除旧 CLI 或兼容脚本。
- 不重写 Python 认证和账号桥接核心。
- 不新增云端服务、遥测或第三方 Swift 依赖。
- 不扩展任务与线程、Hook 与自动化、上下文健康模块。
- 不为了测试故意消耗真实重置卡。

## 验收标准

### AC-PI-001 单一日常 App

- 工作台同时承载主窗口与菜单栏。
- 正常使用时只运行工作台，不运行旧 Profile Switcher。
- 旧 App 保持安装并可在退出工作台后作为冷备启动。

### AC-PI-002 当前登录账号语义

- 菜单栏和账号页主状态只使用 Codex 当前实际登录账号。
- 历史任务账号推断和统计归因不得覆盖当前登录账号。
- 无法确认时显示未知及原因，不显示推断为事实。

### AC-PI-003 菜单栏状态项

- 状态项显示当前账号首要额度百分比和 Codex 工作状态。
- 悬停或可访问性描述包含账号、额度窗口和运行状态。
- 切换、刷新、缓存和错误状态不会显示假零值。

### AC-PI-004 菜单栏展开面板

- 展示当前账号、两档额度、重置卡摘要、快速切换和最近重要操作。
- 提供打开工作台、打开 Codex、刷新、设置和退出。
- 不承载复杂图表或排行。

### AC-PI-005 账号管理完整能力

- 展示账号身份、套餐、托管状态、两档额度和重置时间。
- 展示每张可用重置卡及准确到期时间、缓存和错误状态。
- 展示今日、7 日、14 日账号用量和趋势。
- 其他账号支持“切换并重启 Codex”。
- 任务账号推断和统计归因位于默认折叠的高级诊断。

### AC-PI-006 项目与工具模块

- 项目分析展示项目排行、线程数、token 和最近活动。
- 工具与 Skill 展示调用 / 使用排行及数据健康。
- 两个模块均使用真实后端 payload，不使用静态演示数据。

### AC-PI-007 安全切换与验证

- 切换过程显示明确阶段并禁用竞争动作。
- 继续复用 Python 后端的退出、凭据归档、冲突保护、接管和重启逻辑。
- 切换命令成功后必须重新读取状态；只有实际登录账号等于目标账号且托管正常才算成功。
- 验证失败不写成功日志，并显示目标账号与实际账号。

### AC-PI-008 运行状态一致性

- 使用 Profile Switcher 现有运行状态语义：运行中、待接手、空闲、未知。
- 状态来自对话进程与最近输出证据，不仅检查 Codex App 是否打开。
- 菜单栏、概览和账号页使用同一状态快照。

### AC-PI-009 重置卡通知与自动重置

- 迁移旧工具的到期提醒计划和系统通知。
- 只有官方明确额度耗尽且存在有效卡时才自动消费。
- 到期本身不得触发消费。
- 保留最早到期选择、持久幂等、十分钟重试间隔和终态去重。
- 兼容旧 Profile Switcher 偏好域，冷备切换不会重复消费。

### AC-PI-010 刷新、缓存与错误

- 启动、每 60 秒、系统唤醒和官方额度通知触发统一刷新。
- 刷新失败保留最近成功数据并标注暂存时间。
- 局部错误不清空整个页面；没有可靠值时显示 `--` 和原因。
- 检测到旧 App 正在运行时暂停工作台自动重置并显示冲突提示。

### AC-PI-011 启动行为

- 用户手动打开工作台时显示主窗口。
- 登录 Mac 自动启动时只驻留菜单栏，不主动弹出主窗口。
- 已在后台运行时再次手动打开会显示主窗口。

### AC-PI-012 后端与安装产物

- 工作台包含 Profile Switcher `v0.10.1` 的凭据刷新保护及后续本轮更改。
- 安装指纹覆盖 Swift 源码、资源和实际打包的 Python 后端。
- verifier 能拒绝 Python 后端陈旧或缺失的安装包。

### AC-PI-013 操作日志

- 账号切换、自动重置、重置卡数量变化和账号数据源故障 / 恢复继续记录。
- 事件显示账号、前后状态、来源链和可信度。
- V1.4 上下文摘要与工作流解释回归保持通过。

### AC-PI-014 视觉和可访问性

- 账号、项目、工具页面遵循 `DESIGN.md`，不嵌入旧玻璃卡片 UI。
- 原生窗口在 `900×640`、`1160×780`、`1440×900` 无裁切、越界或横向滚动。
- 菜单栏在浅色和深色可读，状态不只依赖颜色。
- loading、stale、error、switching、unknown 和长文本均有有效截图或可见验证。
- Visual Verdict 总分不低于 90；错误当前账号语义直接失败。

### AC-PI-015 真实迁移验收

- 记录验收开始时账号。
- 已安装工作台能读取两个真实账号的额度与重置卡详情。
- 从初始账号切到另一个账号，再切回并恢复初始账号；两次均验证 Codex 实际登录状态。
- 菜单栏随切换更新正确。
- 验收结束时旧 App 仍安装但不运行。

### AC-PI-016 隐私与安全

- 不输出或持久化 token、Cookie、认证内容或完整重置卡原始 ID。
- 不新增外部网络调用；账号数据继续通过本机 Codex App Server 和本地文件读取。
- 不为测试消费真实重置卡。

## 约束和假设

- 目标平台为 macOS 13 及以上。
- 工作台继续使用 bundle id `com.hd2yao.codex-workbench`。
- 冷备 bundle id 为 `com.hd2yao.codex-profile-switcher`。
- 本机当前存在两个 profile：`hd-master` 与 `hd-sarah-blackwell`；UI 不把此数量硬编码。
- Python 后端是认证与账号切换的权威实现。
- 旧 App 只做冷备，不保证与工作台并行运行。

## 待确认问题

无阻断实现的问题。旧 App 的正式删除和源码归档属于后续独立退役阶段，必须再次由用户确认。
