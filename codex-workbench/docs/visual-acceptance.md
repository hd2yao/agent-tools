## 额度日志专项验收

- `52% → 48%`：只更新观察基线，生成 0 条事件。
- `2% → 0%`：生成“额度已用完”。
- 原定 `resetsAt` 窗口附近恢复到 `>=99%`：生成“额度已按计划刷新”。
- 原定窗口外恢复到 `>=99%` 且无本地重置证据：生成“检测到官方侧额度恢复”，并明确官方未提供原因。
- 重置次数数量变化：保留准确 before/after。
- 单独的 `resetsAt` 变化：不生成事件。
- 旧版 `quota_usage_updated` 与 `quota_reset_time_updated` 噪声启动时原子清理；当前台账两类均为 0 条。
